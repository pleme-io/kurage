# Kurage (クラゲ) — Cursor Cloud Agents CLI + MCP Server

## Build & Test

```bash
cargo build
cargo test --lib
KURAGE_CONFIG=/dev/null cargo run -- me       # test against live API
KURAGE_CONFIG=/dev/null cargo run -- models   # list available models
```

## Architecture

Dual-mode binary: CLI subcommands for terminal use, MCP server (rmcp 0.15 stdio)
when invoked with no arguments. Bridges Cursor's Cloud Agents REST API to any
MCP-aware tool (Claude Code, Cursor IDE, OpenCode, Rovo Dev).

### Modules

| Module | Purpose |
|--------|---------|
| `api/types.rs` | All request/response types — derived from official OpenAPI 3.0.3 spec |
| `client.rs` | `CursorCloudClient` wrapping `todoku::HttpClient` with Bearer auth |
| `auth.rs` | API key resolution: CLI flag > `CURSOR_API_KEY` env > file |
| `config.rs` | `KurageConfig` via `shikumi::ConfigDiscovery` (`~/.config/kurage/kurage.yaml`) |
| `format.rs` | Central text formatting — single source of truth for CLI + MCP output |
| `output.rs` | CLI output dispatch (JSON / Pretty / Table) delegates to `format` |
| `mcp.rs` | 13 MCP tools via `#[tool_router]` (rmcp 0.15) |
| `commands/*.rs` | 12 CLI subcommands (clap 4) |
| `error.rs` | `KurageError` bridging `todoku::TodokuError` |

### Shared Library Usage

- **todoku** — HTTP client + BearerToken auth + retry
- **shikumi** — Config file discovery (XDG paths, env override)
- **kaname** — MCP server scaffold (via rmcp re-exports)

### Nix Integration

- **flake.nix** — substrate `rust-tool-release-flake.nix` + `crateOverrides` for rmcp + `homeManagerModules.default`
- **module/default.nix** — HM module: `services.kurage.{enable, package, mcp, settings}`
- **Dual wiring** — anvil (all agents via envFiles) + service-level (Claude Code via serverEntry + KURAGE_CONFIG env)

### Config

`~/.config/kurage/kurage.yaml` (generated by Nix HM module):
```yaml
api_url: https://api.cursor.com
api_key_file: ~/.config/cursor/api-key
default_model: claude-4.6-opus-high-thinking
output: pretty
poll_interval: 5
```

### OpenAPI Spec

Official spec stored at `specs/cloud-agents-openapi.yaml` for reference and

## Design Decisions

- **Bearer auth** (not Basic) per OpenAPI spec `bearerAuth` scheme
- **Base URL excludes /v0** — the `/v0` prefix is part of each endpoint path
- **`#[serde(flatten)] extra: serde_json::Value`** on all response types for forward compatibility
- **Format module** is the single source of truth — both CLI output.rs and MCP mcp.rs delegate to it
- **MCP tools expose more parameters than CLI** — CLI is simplified, MCP covers full spec surface
